AGENT ARCHITECTURE

双 Agent 判断，
确定性系统裁决。

大模型负责理解语义和提出证据，代码负责验证、打分、限幅与持久化。模型不能直接决定是否报警。

01 · INPUT GATE 不可信新闻输入 限制长度 · 固定字段 · 候选筛选
02 · ANALYST AGENT 语义分析 事件类型 · 强度 · 证据 · 标的链路
03 · SCORE ENGINE 100 分确定性评分 硬门槛 · 权重 · 惩罚 · 上限
04 · CRITIC AGENT 独立质疑 确认 · 降级 · 拒绝 · 删除无依据标的
05 · D1 MEMORY 持久化与市场验证 唯一键去重 · Agent trace · 收盘价回补
PROMPT

证据约束

只能引用标题和摘要中的原文；网页内容被视为不可信数据，不能变成指令。

HARNESS

结构与边界校验

严格 JSON、标的白名单、证据逐字匹配、超时与失败降级；未复核结果最高 59 分。

TOOLS

最小工具权限

QVeris 获取新闻，DeepSeek Agent 不允许调用工具，行情模块只在收盘后获取日线。

MEMORY

可追踪的事件记忆

D1 保存原始新闻、双语文本、评分版本、Agent trace、方向和历史价格反应。

IMPLEMENTATION

一条新闻如何变成
一张信号卡片。

这不是一个黑盒预测器，而是一条可追踪的研究流水线。

  1. 01
    采集新闻

    QVeris 调用 Alpha Vantage 与 Finnhub 能力，按全局、核心标的和完整 universe 分层抓取。

    qveris_client.py
  2. 02
    翻译与分类

    翻译缓存保证中英文一致；规则与大模型共同判断鬼故事类型、可信度与强度。

    agent-harness.ts · scoring.ts
  3. 03
    映射产业链

    从云厂商、芯片、晶圆、内存、服务器、算力租赁到电力与 ETF，逐层给出方向。

    agent-prompts.ts · scoring.ts
  4. 04
    回填市场反应

    反应交易日收盘后拉取日线，计算前收、反应收盘、次日与三交易日表现。

    market-impact.ts · D1
